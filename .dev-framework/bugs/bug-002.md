# BUG-002: Unnecessary nested parens in SettingsPlugin's isSettingsResponse guard

**Severity**: low
**Status**: open
**Created**: 2026-07-04T13:46:56Z

## Description
`isSettingsResponse` in `packages/frontend/src/plugins/settings/SettingsPlugin.tsx` nests
parentheses more than needed around the `orientation` check. Flagged by the Simplify agent during
the `additional-features` reviewer pass.

## Steps to Reproduce
Read `packages/frontend/src/plugins/settings/SettingsPlugin.tsx`, the `isSettingsResponse`
function.

## Expected Behavior
N/A — cosmetic only, no behavior difference.

## Actual Behavior
Extra grouping parens around the `'orientation' in value && (...)` clause.

## Root Cause
Written for clarity during initial implementation, slightly over-grouped.

## Fix
Flatten the boolean expression next time this file is touched. Not urgent enough to justify a
standalone change.
