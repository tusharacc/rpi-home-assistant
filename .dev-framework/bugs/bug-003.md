# BUG-003: News SourceMenu popover has no click-outside-to-close handler

**Severity**: low
**Status**: open
**Created**: 2026-07-05T06:45:00Z

## Description
`SourceMenu` in `packages/frontend/src/plugins/news/OtherNews/SourceMenu.tsx` opens a small
popover (Follow Source / Hide Source) on click, but nothing closes it when the user clicks
elsewhere on the page. Flagged during the `news-intelligence` reviewer pass.

## Steps to Reproduce
Open the News plugin's "Other News" tab, click the overflow (⋮) icon on any article card,
then click anywhere else on the card list.

## Expected Behavior
The popover should close when the user clicks outside of it.

## Actual Behavior
The popover stays open until the same trigger button is clicked again.

## Root Cause
No outside-click listener was wired up; `open` state only toggles on the trigger button itself.

## Fix
Add a document-level click listener (or a lightweight outside-click hook) that closes the
popover when the click target is outside its DOM node. Not urgent — touch-only kiosk usage
means the user can just tap the trigger again to dismiss it.
