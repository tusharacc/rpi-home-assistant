# BUG-004: No manage-sources view — SourceMenu fires hide/follow blind

**Severity**: low
**Status**: open
**Created**: 2026-07-05T06:45:00Z

## Description
The backend exposes `GET /api/news/sources` (list of sources with `hidden`/`followed` flags,
see `packages/backend/src/routes/news.ts`), but no frontend component ever calls it.
`SourceMenu.tsx` fires `POST /api/news/sources/:name/hide` and `/follow` without knowing or
showing the source's current state, and there's nowhere in the UI to review or undo a hide/follow
after the fact. Flagged during the `news-intelligence` reviewer pass; the developer log already
called this out as an intentionally deferred follow-up rather than an oversight.

## Steps to Reproduce
Hide a source via the article card's overflow menu, then look for any way to see or un-hide it.
There isn't one.

## Expected Behavior
A "manage sources" view (e.g. a settings row or dedicated panel) listing all known sources with
their hidden/followed state and a way to toggle each.

## Actual Behavior
Hide/follow actions are one-way from the user's perspective in the UI (reversible only via a
direct API call).

## Root Cause
Out of scope for the initial PO requirements, which specified the Hide Source/Follow Source
*actions* but not a management view for reviewing prior choices.

## Fix
Add a small source-list panel (could live under Settings or as a News sub-view) that fetches
`GET /api/news/sources` and renders toggle controls per source.
